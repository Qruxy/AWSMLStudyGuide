7. Machine Learning Concepts

Accuracy

A measure of how often the model correctly predicts the outcome.

  • Calculated as (True Positives + True Negatives) / Total Predictions
  • Useful for balanced datasets
  • Can be misleading for imbalanced datasets

Example: A model that correctly predicts 90 out of 100 test samples has 90% accuracy.

Precision and Recall

Key metrics used to evaluate classification models.

  • Precision: The ratio of true positives to the sum of true and false positives.
  • Recall: The ratio of true positives to the sum of true positives and false negatives.
  • Balance between precision and recall is crucial.

Example: Evaluating a spam detection model where high precision reduces false alarms.

Confusion Matrix

A table used to describe the performance of a classification model.

  • Shows true positives, false positives, true negatives, and false negatives
  • Helps in calculating precision, recall, and accuracy
  • Useful for multi-class classification problems

Example: Analyzing model errors by examining the confusion matrix.

Overfitting and Underfitting

Concepts that describe how well a model generalizes to unseen data.

  • Overfitting: Model learns noise and performs poorly on new data.
  • Underfitting: Model is too simple and cannot capture underlying patterns.
  • Balancing complexity is key.

Example: A complex model that performs well on training data but poorly on test data is overfitting.

External configurations set before training a model.

  • Examples include learning rate, number of layers, and regularization parameters
  • Affect how the model learns
  • Optimized through techniques like grid search or random search
